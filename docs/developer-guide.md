# UniFlow Developer Guide

## Introduction
UniFlow is an open-source project that welcomes contributors who want to improve student productivity tools.

## Repository Structure
- `index.html` : main page
- `users.html` : user page
- `contributors.html` : contributors page
- `README.md` : project overview
- `CODE_OF_CONDUCT.md` : contributor behavior rules
- `CONTRIBUTING.md` : contribution process
- `docs/` : project documentation
- `screenshots/` : screenshots and visual materials
- `tasks/` : team task distribution
- `system-info/` : project and module information
- `src/` : source files and future implementation files

## Contribution Process
1. Fork the repository.
2. Create a new branch for your change.
3. Make your changes clearly and carefully.
4. Test your work before submission.
5. Submit a pull request with a proper explanation.

## Coding Guidelines
- Use meaningful file and section names.
- Keep code organized and readable.
- Follow the project structure.
- Write clear commit messages.

## Communication
Contributors can communicate through:
- GitHub Issues
- GitHub Discussions
- Pull Requests

## Development Goal
The goal of UniFlow is to provide a maintainable and useful open-source platform for university students.
